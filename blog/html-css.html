Best Practices Associated with Classes vs Ids

Classes and Ids are attributes used to apply interactivity or styling to a block, often defined by a div or span tag. While in function they are very similar, it’s very important to note that the application for each of them is different. As such, it’s important to learn the contexts in which they should be used, and how one may benefit over the other in certain scenarios.

An important difference between ids and classes is that ids are specific to one single element and should only be used once on a page. A class on the other hand can be used for multiple elements and can be used as many times as necessary. You can even assign one element several classes. An easy analogy for this is to think of the id as a student id and the class as actual classes. A student id is specific to one student, and that student can only have one id. However that student could take multiple classes, and classes can have multiple students.

One last thing that should be mentioned is that ids have higher priority in your code than classes. If an element has a class and an id assigned to it with conflicting instructions, the id will take precedence.

So what are the best practices associated with using classes vs ids? In general I would argue that it’s best to use classes when you can. Classes are flexible, and if you find yourself wanting to use your class on an element later on in your coding journey, you can do this with no problems. Recurring blocks on your page, such as paragraphs, headers, and buttons, should always be assigned a class attribute. However if you had one element on your page that didn’t recur anywhere else and used a particular style or interactiveness, using an id would be suitable.
